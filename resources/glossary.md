# CISSP Glossary

80+ essential terms. Review this weekly — terminology is tested directly and in scenario questions.

---

## A

| Term | Definition |
|------|-----------|
| **Access Control** | Security features that control how users and systems interact with resources |
| **Accountability** | Recording subject actions so they can be attributed to an individual |
| **ALE** | Annualised Loss Expectancy — expected annual cost of a threat (SLE × ARO) |
| **ARO** | Annualised Rate of Occurrence — how often a threat is expected per year |
| **Asset** | Anything of value to an organisation (data, hardware, people, reputation) |
| **Assurance** | Degree of confidence that security controls are effective |
| **Asymmetric Encryption** | Uses key pairs (public + private); slower but solves key distribution |
| **Audit Trail** | Chronological record of system activities for review and forensics |
| **Authentication** | Proving the identity you claimed during identification |
| **Authorisation** | Determining what an authenticated subject is permitted to do |
| **Availability** | Ensuring systems and data are accessible when needed |

## B

| Term | Definition |
|------|-----------|
| **Baseline** | Minimum security configuration for a specific platform |
| **Bell-LaPadula** | Confidentiality model: no read up, no write down |
| **Biba** | Integrity model: no read down, no write up |
| **BCP** | Business Continuity Plan — keeps business running during disruptions |
| **Biometrics** | Authentication using physical characteristics |

## C

| Term | Definition |
|------|-----------|
| **CA** | Certificate Authority — issues and signs digital certificates |
| **CAT** | Computerised Adaptive Testing — exam adapts to your responses |
| **CIA Triad** | Confidentiality, Integrity, Availability — foundation of security |
| **CER** | Crossover Error Rate — where FAR equals FRR; lower = more accurate |
| **Chain of Custody** | Documentation tracking evidence handling from collection to court |
| **Clark-Wilson** | Commercial integrity model using transactions and separation of duties |
| **Clipping Level** | Threshold of violations before an alert is triggered |
| **Cold Site** | DR site that is a shell building — weeks to bring online |
| **Confidentiality** | Preventing unauthorised disclosure of information |
| **Countermeasure** | A control that reduces a risk |
| **CRL** | Certificate Revocation List — list of invalid/revoked certificates |

## D

| Term | Definition |
|------|-----------|
| **DAC** | Discretionary Access Control — resource owners set permissions |
| **Data Custodian** | Implements and maintains controls defined by the Data Owner |
| **Data Owner** | Senior manager responsible for classification and use decisions |
| **Degaussing** | Using a magnetic field to destroy data on magnetic media |
| **DLP** | Data Loss Prevention — monitors and blocks unauthorised data movement |
| **DRM** | Digital Rights Management — controls what recipients can do with data |
| **DRP** | Disaster Recovery Plan — restores IT systems after a disaster |
| **Due Care** | Taking responsible action based on known risks |
| **Due Diligence** | Investigating and understanding risks before acting |

## E

| Term | Definition |
|------|-----------|
| **EF** | Exposure Factor — percentage of asset value lost per incident |
| **Encryption** | Transforming data to protect it from unauthorised access |
| **Enumeration** | Phase 2 of pen testing — port scanning and resource identification |

## F

| Term | Definition |
|------|-----------|
| **FAR** | False Acceptance Rate (Type II error) — impostor is accepted |
| **FRR** | False Rejection Rate (Type I error) — valid user is rejected |
| **Firewall** | Network device that enforces access control rules |

## G

| Term | Definition |
|------|-----------|
| **Guideline** | Recommended (not mandatory) security practice |

## H

| Term | Definition |
|------|-----------|
| **Hashing** | One-way function producing fixed-length output — protects integrity |
| **Hot Site** | Fully operational DR site — can be online within hours |

## I

| Term | Definition |
|------|-----------|
| **Identification** | Claiming an identity (e.g. entering a username) |
| **Integrity** | Ensuring information is accurate and has not been modified |
| **IDS** | Intrusion Detection System — monitors for suspicious activity |
| **IPS** | Intrusion Prevention System — monitors and actively blocks |

## K

| Term | Definition |
|------|-----------|
| **KDC** | Key Distribution Centre — central component of Kerberos |
| **Kerberos** | Ticket-based authentication protocol using symmetric cryptography |

## L

| Term | Definition |
|------|-----------|
| **Least Privilege** | Give subjects only the access needed to perform their job |

## M

| Term | Definition |
|------|-----------|
| **MAC** | Mandatory Access Control — OS enforces access based on labels |
| **MFA** | Multi-Factor Authentication — 2+ different factor types required |
| **MTD** | Maximum Tolerable Downtime — absolute limit before business impact is critical |

## N

| Term | Definition |
|------|-----------|
| **Need-to-Know** | Access granted only when required for job function |
| **Non-repudiation** | Proof that a specific entity sent or created a message (digital signatures) |

## P

| Term | Definition |
|------|-----------|
| **PKI** | Public Key Infrastructure — framework for issuing/managing digital certificates |
| **Policy** | Senior management's high-level security intentions and direction |
| **Procedure** | Step-by-step instructions for performing a specific security task |

## R

| Term | Definition |
|------|-----------|
| **RBAC** | Role-Based Access Control — permissions assigned to roles, not individuals |
| **Residual Risk** | Risk remaining after controls are applied |
| **Risk** | Probability that a threat agent will exploit a vulnerability |
| **RPO** | Recovery Point Objective — maximum acceptable data loss (time-based) |
| **RTO** | Recovery Time Objective — maximum acceptable downtime |

## S

| Term | Definition |
|------|-----------|
| **Safeguard** | A control that reduces the probability or impact of a threat |
| **Separation of Duties** | No single person can complete a critical transaction alone |
| **SIEM** | Security Information and Event Management — log aggregation and alerting |
| **SLE** | Single Loss Expectancy — cost if a threat occurs once |
| **SSO** | Single Sign-On — one credential grants access to multiple systems |
| **Standard** | Mandatory rule that supports and implements a security policy |

## T

| Term | Definition |
|------|-----------|
| **TCB** | Trusted Computing Base — all protection mechanisms in a system |
| **Threat** | Any potential danger to an asset |
| **Threat Agent** | Entity that carries out a threat (hacker, disgruntled employee, hurricane) |
| **TGT** | Ticket Granting Ticket — Kerberos credential for requesting service tickets |

## V

| Term | Definition |
|------|-----------|
| **Vulnerability** | A weakness that a threat agent can exploit |

## W

| Term | Definition |
|------|-----------|
| **Warm Site** | DR site with hardware but needing configuration — days to bring online |
